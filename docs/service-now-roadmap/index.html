Details

Single Sign-On Gateway (BCeID)

Users can access the ServiceNow catalogue in a secure way using their BCeIDs

Service Desk Process Discovery

Users will access ServiceNow to request 17 distinct services - how should this work? We explore much of that in this phase.

Service Desk Process Development

We select a sub-set of service requests to implement and iterate upon. Lessons learned here will inform subsequent efforts.

ISPO Intake Discovery

ISPO seeks a single front-door and work organization system for managing their services. We'll use this time to investigate suitability and priorities.

Tokyo Upgrade

Update ensures continued security and stability of the platform; makes new functionality available; and allows us to adopt a refreshed user interface.

User Interface Updates

The Now Platform UI design system refines and modernizes UI elements - this time is spent ensuring the transition is a smooth one and users are consulted.

Documentation Review

Consolidates and qualifies documentation, improving discoverability of reference resources. This is a first step in ensuring we have a high-quality repository of evergreen resources for new users and teams.

Training Video Inventory

We have a large, but unstructured collection of videos from training sessions. This work catalogues thos videos and trims them to their core content.

ServiceNow Intake Design

Ensures a clear process and readiness criteria for onboarding to ServiceNow, requesting enhancements, or soliciting new product development.

Linkages

Each work item supports one or more facets of the product vision:

A single unified service delivery platform enabling ISD as a collaborative and empowered team to provide exceptional user experience and achieve operational excellence.
